# Workflow: 일상 비서 (캘린더 · 좌석 모니터링 · 아카이빙)

## INPUT
- 캘린더 접근 권한 (Google Calendar 등)
- 모니터링 대상 (KTX/SRT 구간·날짜·시간대, 관심 좌석 등급)
- 아카이빙 대상 (카카오톡 '나에게 보내기' 내보내기 파일 등 — kakao-archive 웹앱과 연계)

## PROCESS

### A. 캘린더 관리
1. 새 일정 요청 시 기존 일정과 충돌 여부 먼저 확인
2. 반복 일정/미팅은 사람 승인 없이 바로 등록 가능, 단 참석자가 있는 미팅은 등록 전 확인
3. 하루 시작 시 "오늘 일정 요약"을 사람에게 먼저 보고 (예약 작업으로 걸어두면 자동)

### B. KTX/SRT 좌석 모니터링
1. 구간/날짜/시간대/좌석등급 조건을 명확히 저장
2. 주기적으로(예: 10~30분 간격) 조회 → 조건에 맞는 좌석 발견 시 즉시 알림
   ⚠️ 예매 자체는 결제·개인정보가 걸리므로 **자동 예매하지 않고 알림까지만** 수행 (사람이 직접 예매)
3. 알림에는 열차편/시간/좌석등급/잔여석을 명시

### C. 나에게 보내기 아카이빙
1. `kakao-archive` 웹앱과 동일한 파싱 규칙 사용 (링크/할일/사진/메모 자동 분류)
2. 주기적으로 내보내기 파일을 받으면 자동으로 파싱해서 정리된 형태로 다시 보고

## VALIDATE
- [ ] 캘린더 충돌이 있는 일정을 사람 확인 없이 등록하지 않았는가
- [ ] 좌석 모니터링이 "알림"에서 멈추고 자동 결제로 넘어가지 않았는가
- [ ] 모니터링 주기가 과도한 요청(차단 위험)을 유발하지 않는가
