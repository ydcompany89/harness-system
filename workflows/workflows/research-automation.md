# Workflow: 리서치 & 영업 자동화

## INPUT
- 모니터링 대상 키워드/도메인 리스트
- 이메일 수신함 (분류 대상)
- 웹 크롤링 대상 URL 패턴

## PROCESS

### A. 뉴스 모니터링
1. 키워드 리스트 기준 web_search 실행 (하루 1~2회 주기 권장)
2. 중복/노이즈 제거 → 관련도 순 정렬
3. 3줄 요약 + 원문 링크로 정리 (본문 재현 금지, 저작권 준수)

### B. 이메일 분류 및 요약
1. 카테고리 정의 (예: 긴급/영업/뉴스레터/스팸)
2. 카테고리별 규칙을 이 파일에 계속 추가 (오분류 발견 시마다 업데이트)
3. 요약은 "발신자 / 핵심 요청사항 / 마감기한" 3항목 고정 포맷

### C. 세금계산서 발행
- ⚠️ 금액/거래처 정보는 사람이 최종 승인 후 발행 (자동 발행 금지 권장)
1. 거래 내역 입력받기
2. 계산서 초안 생성
3. 사람 승인 → 발행

### D. 웹 크롤링
1. 대상 URL 패턴 정의
2. `web_fetch` 로 수집 (robots.txt 준수, 과도한 요청 금지)
3. 구조화(표/CSV)해서 저장

## VALIDATE
- [ ] 뉴스 요약에 원문 그대로 복사된 문장 없는가 (패러프레이즈 확인)
- [ ] 이메일 분류 정확도가 떨어지면 카테고리 규칙 즉시 갱신했는가
- [ ] 금액이 걸린 작업(세금계산서)은 사람 승인 단계를 거쳤는가
