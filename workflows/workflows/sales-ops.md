# Workflow: 영업 & 고객 응대

## INPUT
- 리드/문의 이메일 또는 폼 제출 내용
- 기존 고객 히스토리 (CRM, 이전 대화 요약)
- 가격/제안서 템플릿

## PROCESS

1. **리드 분류**: 문의 내용을 (진지한 구매의사 / 정보성 문의 / 스팸)으로 분류
2. **1차 응답 초안**: 톤은 정중하되 다음 액션을 명확히 제시 (미팅 제안, 자료 발송 등)
3. **제안서/견적서**: 템플릿 기반으로 고객명·요구사항만 치환, 숫자(가격)는 반드시 사람이 최종 확인
4. **후속(팔로업)**: 응답 없는 리드는 N일 후 자동 리마인드 초안 생성 (발송은 사람이 승인)

## VALIDATE
- [ ] 가격/계약 조건이 들어간 문서는 발송 전 사람 승인을 거쳤는가
- [ ] 고객별 히스토리를 반영해 톤이 어긋나지 않았는가
- [ ] 팔로업 주기가 스팸처럼 느껴지지 않게 적절한가 (권장: 3~5일 간격)
